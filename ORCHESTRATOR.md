# ORCHESTRATOR

## Назначение

Отвечает за управление процессом создания поста и взаимодействием между компонентами архитектуры.

ORCHESTRATOR не участвует в создании, проверке или изменении текста поста.

---

## Ответственность

ORCHESTRATOR отвечает только за управление процессом выполнения.

ORCHESTRATOR:

- получает запрос пользователя;
- запускает `WORKFLOW_WRITE_POST`;
- передает Workflow необходимые входные данные;
- получает результат выполнения Workflow;
- передает результат в `OUTPUT_FORMAT`;
- рассматривает результаты работы компонентов как входные данные и не оценивает их содержимое;
- передает данные между компонентами без изменений.

ORCHESTRATOR не изменяет текст поста.

ORCHESTRATOR не проверяет качество текста.

ORCHESTRATOR не принимает решений о содержании поста.

ORCHESTRATOR не изменяет требования `POST_STANDARD`.

---

## Input

- запрос пользователя.

---

## Процесс выполнения

1. Получить запрос пользователя.
2. Запустить `WORKFLOW_WRITE_POST`.
3. Передать Workflow:
   - запрос пользователя.
4. Получить результат от Workflow.
5. Передать результат в `OUTPUT_FORMAT`.

---

## Недопустимо

ORCHESTRATOR не должен:

- писать текст поста;
- изменять текст поста;
- проверять качество текста;
- выполнять функции Writer;
- выполнять функции Critic;
- изменять требования `POST_STANDARD`;
- изменять запрос пользователя;
- изменять замечания Critic перед передачей Writer;
- принимать решения о содержании поста;
- изменять результат Workflow перед передачей в `OUTPUT_FORMAT`.

---

## Output

- результат Workflow, переданный в `OUTPUT_FORMAT` без изменений.
