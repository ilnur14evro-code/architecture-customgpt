# WORKFLOW_WRITE_POST

## Назначение

WORKFLOW_WRITE_POST управляет процессом создания поста.

Workflow управляет взаимодействием между Writer и Critic до завершения процесса.

---

## Ответственность

Workflow отвечает только за управление процессом создания поста.

Workflow:

- запускает Writer и Critic;
- передает результаты между ролями;
- контролирует завершение Workflow.

Workflow не пишет текст.

Workflow не проверяет качество текста.

Workflow не изменяет требования `POST_STANDARD`.

Workflow не изменяет запрос пользователя.

Workflow не принимает решений о содержании поста.

---

## Input

- запрос пользователя.

---

## Процесс выполнения

1. Запустить Writer.
2. Передать Writer:
   - запрос пользователя;
   - `POST_STANDARD`.
3. Получить текущую версию поста от Writer.
4. Передать Critic:
   - текущую версию поста;
   - запрос пользователя;
   - `POST_STANDARD`.
5. Получить результат проверки от Critic.
6. Если Critic вернул статус `REVISION_REQUIRED`:
   - передать Writer текущую версию поста;
   - передать Writer замечания Critic;
   - повторить процесс проверки.
7. Если Critic вернул статус `APPROVED`:
   - завершить Workflow;
   - вернуть последнюю версию поста ORCHESTRATOR.
8. Если достигнуто максимальное количество итераций (`MAX_ITERATIONS = 5`) до получения статуса `APPROVED`:
   - завершить Workflow;
   - вернуть последнюю версию поста ORCHESTRATOR.

---

## Completion Criteria

Workflow считается завершенным после:

- получения статуса `APPROVED` от Critic;

или

- достижения максимального количества итераций (`MAX_ITERATIONS = 5`).

Получение статуса `APPROVED` считается успешным завершением Workflow.

---

## Недопустимо

Workflow не должен:

- создавать текст поста;
- изменять текст поста;
- проверять качество текста;
- изменять требования `POST_STANDARD`;
- изменять запрос пользователя;
- изменять замечания Critic;
- выполнять функции Writer;
- выполнять функции Critic;
- продолжать выполнение после достижения критериев завершения Workflow.

---

## Output

- последняя версия поста.
