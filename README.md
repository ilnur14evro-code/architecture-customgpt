# Custom GPT для написания постов

## О проекте

Проект представляет собой модульную архитектуру Custom GPT, предназначенную исключительно для написания качественных постов для социальных сетей.

Архитектура построена по принципу разделения ответственности. Каждый компонент выполняет только одну функцию и не дублирует обязанности других компонентов.

---

# Назначение проекта

Цель проекта — обеспечить предсказуемое и воспроизводимое создание постов за счет четкого разделения процесса на независимые компоненты.

Архитектура не является универсальным фреймворком и не предназначена для решения задач, не связанных с написанием постов.

# Структура проекта

```text

WORKFLOW_WRITE_POST.md

ROLE_CRITIC.md
POST_STANDARD.md
OUTPUT_FORMAT.md
```

---

# Назначение файлов

| Файл | Назначение |
|------|------------|
| `SYSTEM.md` | Определяет общие правила работы GPT. |
| `ARCHITECTURE.md` | Описывает архитектуру проекта и взаимодействие компонентов. |
| `ORCHESTRATOR.md` | Управляет выполнением процесса. |
| `WORKFLOW_WRITE_POST.md` | Определяет процесс создания поста. |
| `ROLE_WRITER.md` | Создает и дорабатывает текст поста. |
| `ROLE_CRITIC.md` | Проверяет текст поста. |
| `POST_STANDARD.md` | Определяет требования к качеству постов. |
| `OUTPUT_FORMAT.md` | Отображает результат пользователю без изменений. |

# Основные принципы

- Один компонент — одна ответственность.
- Все компоненты выполняют только собственные функции.
- `POST_STANDARD` является единственным источником требований к качеству постов.
- ORCHESTRATOR управляет процессом, но не создает и не проверяет текст.
- WORKFLOW управляет взаимодействием между ролями.
- Writer создает и дорабатывает текст.
- Critic проверяет текст.
- OUTPUT_FORMAT только отображает результат.
- SYSTEM определяет только общие правила работы GPT.

---

# Изменение проекта

При изменении проекта необходимо соблюдать следующие правила:

- сохранять разделение ответственности между компонентами;
- не переносить обязанности одного компонента в другой;
- не дублировать требования между файлами;
- изменять требования к качеству постов только в `POST_STANDARD.md`;
- изменять процесс создания постов только в `WORKFLOW_WRITE_POST.md`;
- изменять обязанности компонентов только в соответствующих файлах.
